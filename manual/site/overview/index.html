Overview

Log(ML) is a Machine Learning / AI automation framework. Here is a summary of what LogMl does:

Example of LogMl workflow:

  1. Dataset:
    1. Load: Load a dataset from a file
    2. Transform: sanitize input names, encode cathegorical inputs, one-hot encoding, expand date/time fieds, remove duplicated inputs, remove samples with missing outputs, shuffle samples, remove inputs with low variance, impute missing values, add missing value indicator variables, normalize inputs, etc.
    3. Augment: add pricipal components (PCA), NMF, interaction variables, etc.
    4. Split: Split data into train / validate / test datasets
    5. Inputs/Outputs: Obtain inputs and output for each (train / validate / test) dataset.
  2. Explore: show dataset, correlation analysis, dendogram, missingness analysis, normality/log-normality analysis, inputs distributions, pair-plots, heatmaps, etc.
  3. Feature importance:
    1. Model-based (Random Forest, Extra trees, Gradient Boosting) by input shuffle / drop column algorithms
    2. Boruta
    3. Regularization methods (Lasso, Ridge, Lars)
    4. Recursive Feature elimination (Lasso, Ridge, Lars AIC, Lars BIC, Random Forest, Extra trees, Gradient Boosting)
    5. Linear model p-value
    6. Logistic model p-value (Wilks)
    7. Multi-class Logistic model p-value (Wilks)
    8. Summary by weighted consensus of all the previous method using model's validation score.
  4. Model Training:
    1. Single model: Train a model, calculate validation and test dataset performance
    2. Hyper-parameter optimization: Train models using hyper-parameter optimization using Bayesian or Random algorithms
    3. Model Search: Search over 50 different model families (i.e. machine learning algorithms). This search can be done using hyper-parameter optimizations on each model's hyper-parameters.
  5. Cross-validation: Perform cross-validation (KFold, RepeatedKFold, LeaveOneOut, LeavePOut, ShuffleSplit). Cross-validation can be used in all model search and Feature importance steps.
  6. Logging: All outputs are directed to log files
  7. Save models & datasets: Automatically save models and datasets for easier retrieval later
